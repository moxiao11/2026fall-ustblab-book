# 分文件编写要点

不知道你在写实验1的时候是什么感受，感觉300行的代码从头写到尾部会很难受，而且感觉很累赘，盯着唯一的main.cpp感觉心碎了一地。

当程序只有几十行时，所有代码写在一个 `main.cpp` 里还能接受。

但当程序越来越大，把不同职责的代码拆到不同文件里，会更容易阅读、测试和修改。

本实验使用 5 个源码文件：

- `main.cpp`
- `TypingTest.h`
- `TypingTest.cpp`
- `TypingConsole.h`
- `TypingConsole.cpp`

---

## 1. 每个文件负责什么

### main.cpp

`main.cpp` 负责主流程。

它不关心“WPM 怎么算”，也不关心“计时器怎么实现”，只负责把步骤串起来：

```cpp
console.showHeader();
console.showPrompt(target);
console.startTimer();
string typed = console.readTypedLine();
double seconds = console.stopTimer();
TypingStats stats = buildStats(target, typed, seconds);
console.showResult(stats);
```

这样主函数读起来就像程序的流程说明。

### TypingTest.h

`.h` 文件放声明。

`TypingTest.h` 告诉其他文件：

- 有哪些常量
- 有哪些函数
- `TypingStats` 结构体长什么样

例如：

```cpp
struct TypingStats {
    double seconds;
    int correctChars;
    int errors;
    double wpm;
    double accuracy;
};

int countCorrectChars(const string& target, const string& typed);
```

### TypingTest.cpp

`.cpp` 文件放实现。

`TypingTest.cpp` 负责真正写出这些函数怎么工作：

```cpp
int countCorrectChars(const string& target, const string& typed) {
    // implementation here
}
```

### TypingConsole.h

`TypingConsole.h` 声明一个类：

```cpp
class TypingConsole {
public:
    void showHeader() const;
    void startTimer();
    double stopTimer() const;

private:
    chrono::steady_clock::time_point startTime;
};
```

### TypingConsole.cpp

`TypingConsole.cpp` 实现这个类的成员函数。

实现成员函数时，需要加上类名和 `::`：

```cpp
void TypingConsole::showHeader() const {
    cout << "Typing Speed Test" << endl;
}
```

## 2. include 的作用

如果一个 `.cpp` 文件想使用某个头文件里的声明，就要 `#include` 它。

例如 `main.cpp` 里要使用 `TypingConsole` 和 `TypingStats`：

```cpp
#include "TypingConsole.h"
#include "TypingTest.h"
```

自己写的头文件用双引号：

```cpp
#include "TypingTest.h"
```

标准库头文件用尖括号：

```cpp
#include <iostream>
```

---

## 3. 头文件保护

你会在 `.h` 文件里看到这样的代码：

```cpp
#ifndef TYPING_TEST_H
#define TYPING_TEST_H

// declarations

#endif
```

这叫 include guard，作用是防止同一个头文件被重复包含。

如果没有 include guard，复杂项目里很容易出现重复定义问题。

---

## 4. 声明和定义的区别

声明只是告诉编译器“有这个东西”：

```cpp
int countErrors(const string& target, const string& typed);
```

定义才是真正写出函数内容：

```cpp
int countErrors(const string& target, const string& typed) {
    int errors = 0;
    return errors;
}
```

一般来说：

- 函数声明放 `.h`
- 函数定义放 `.cpp`
- 类的声明放 `.h`
- 类成员函数的定义放 `.cpp`

---

## 5. 多文件编译

多文件程序编译时，要把所有 `.cpp` 文件都交给编译器：

```bash
g++ -std=c++11 -o typing main.cpp TypingTest.cpp TypingConsole.cpp
```

如果只编译：

```bash
g++ -std=c++11 -o typing main.cpp
```

可能会遇到类似这样的错误：

```text
undefined reference to `chooseSentenceIndex()'
undefined reference to `TypingConsole::showHeader()'
```

这不是函数名写错，而是实现这些函数的 `.cpp` 文件没有参与编译。

---

## 6. 常见错误

### 忘记 include 头文件

如果看到：

```text
TypingConsole was not declared in this scope
```

检查 `main.cpp` 有没有：

```cpp
#include "TypingConsole.h"
```

### 成员函数忘记写类名

错误写法：

```cpp
void showHeader() const {
}
```

正确写法：

```cpp
void TypingConsole::showHeader() const {
}
```

### 只编译了 main.cpp

多文件项目必须编译所有 `.cpp`：

```bash
g++ -std=c++11 -o typing main.cpp TypingTest.cpp TypingConsole.cpp
```

---

## 7. 为什么这样拆

这个实验里：

- `TypingTest` 负责“计算”
- `TypingConsole` 负责“和用户交互”
- `main.cpp` 负责“流程”

这就是一种简单的职责分离。

以后程序更大时，你可以继续按职责拆分文件，而不是把所有东西都塞进一个很长的 `main.cpp`。
