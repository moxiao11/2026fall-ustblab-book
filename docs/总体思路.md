# 设计实验的初衷

设计这组实验的初衷，是把见过的、学过的、有用的知识传递出去。有些知识很难，有些代码 AI 就能写，但我们仍需要不断丰盈自己的知识体系——这才是实验的意义。

## 实验设计思路

**Lab1**：大多数人学编程是为了做一款属于自己的游戏。刚开始时，大家习惯把代码全部扔进一个文件里——能跑就行，没有结构意识。Lab1 从这里切入，让你在实现一个小游戏的过程中，体会"分块思考"的编程习惯。

**Lab2**：你开始学习分文件编写，接触头文件、随机数、计时，把网上学到的小游戏 demo 真正组织起来。这个实验致敬了 CS61A 的打字小实验，但篇幅更长、更完整。

**Lab3**：逐步接触高级语法——泛型、vector、运算符重载，这些课堂上觉得"学了不知道怎么用"的内容，在这里有了用武之地。简易的numpy再现数据科学的魅力，或许将来你走上机器科学的道路，测试文件让你体会一下单元测试的思想，或许你也会成为TDD开发的一员。

**Lab4**: 代码文件的数量进一步增加，你学习了抽卡算法，揭秘了一个个奖池背后的逻辑，不再执着于g++一系列的文件，而是选用了一种快捷的方法，make选定你想要编译运行的文件。

**Lab5**: 你不再执着于写泛型编程，而是学习了一种用弦波构造乐谱的美妙方法，这一次你动手写了makefile、或许也有测试文件，但真正收货颇丰的是map和文件的操作。这个实验致敬了CS61B的音乐项目，可惜能力有限，未能再现其妙处。

实验手册分为以下几个部分：

- [语法学习](./lab5/01-语法学习.md) — 文件读写、关联容器与常用技巧
- [声音合成方法](./lab5/02-声音合成方法.md) — 频率、正弦波、谐波与 ADSR
- [生成的 WAV 文件](./lab5/03-生成的wav文件.md) — 格式、写入与播放
- [实验任务](./lab5/04-实验任务.md) — TODO 任务清单与编译运行

## 希望在实验中带走的知识

### Lab1
- C++ 的条件、循环、数组、函数、结构体等（必会）
- 控制台操作（了解）
- 封装版块的思想（必会）
- 命令行编译（必会）

### Lab2
- C++ 中类与对象（必会）
- 随机数的生成、计时（了解）
- 分文件编写，能写一份安全的头文件（必会）
- 命令行编译（必会）

### Lab3
- C++11 新语法、vector 的使用（必会）
- 模板、运算符重载（了解）
- NumPy 库的介绍（了解）
- 测试驱动开发——编写简单的测试文件（必会）

### Lab4 
- 异常处理、模板的分文件编写(了解)
- 分层抽卡算法(了解)
- 阅读makefile(必会)

### Lab5 

- map的使用、流读写文件的方式、细节小语法(必会)
- 音乐合成算法(了解)
- 动手编写makefile(必会)
- 学习乐谱(你说呢？孩子，你是学计算机的)
