# 控制台操作（分平台说明）

本实验需要操控终端画面——定位光标、设置颜色、非阻塞读键盘。这些操作在不同操作系统上的实现完全不同。下面分 Windows 和 Mac/Linux 两套方案来讲。

> 建议先确认自己的平台，然后只看对应的小节。

---

## Windows 方案

### 坐标定位：gotoxy

```cpp
#include <windows.h>

void gotoxy(int x, int y) {
    COORD coord;
    coord.X = x;
    coord.Y = y;
    SetConsoleCursorPosition(GetStdHandle(STD_OUTPUT_HANDLE), coord);
}
```

控制台原点 `(0, 0)` 在左上角，x 往右，y 往下。

### 隐藏光标

```cpp
void hideCursor() {
    HANDLE hOut = GetStdHandle(STD_OUTPUT_HANDLE);
    CONSOLE_CURSOR_INFO info;
    GetConsoleCursorInfo(hOut, &info);
    info.bVisible = false;
    SetConsoleCursorInfo(hOut, &info);
}
```

### 设置颜色：setColor

```cpp
void setColor(int color) {
    SetConsoleTextAttribute(GetStdHandle(STD_OUTPUT_HANDLE), color);
}
```

Windows 颜色代码：

| 代码 | 颜色 | 画什么 |
|------|------|--------|
| 2 | 深绿 | 蛇身 `#` |
| 7 | 白色 | 边框 |
| 10 | 绿色 | 蛇头 `@` |
| 12 | 红色 | 食物 `$` |
| 14 | 黄色 | 分数 |

### 延时：Sleep

```cpp
#include <windows.h>
Sleep(150);   // 暂停 150 毫秒
```

### 非阻塞键盘输入

```cpp
#include <conio.h>

if (_kbhit()) {          // 有按键吗？
    int key = _getch();  // 读取按键值
}
```

### 方向键

方向键是两个字节的序列：第一个是 `0xE0`，第二个是方向码（↑72 ↓80 ←75 →77）。需要连续调两次 `_getch()`。

## Mac / Linux 方案

Mac 和 Linux 的终端都支持 **ANSI 转义序列**，一套标准来控制光标和颜色，不需要任何平台专属的头文件。

### 坐标定位

用 ANSI 转义序列 `\033[y;xH`：

```cpp
#include <iostream>
using namespace std;

void gotoxy(int x, int y) {
    cout << "\033[" << y << ";" << x << "H";
}
```

> 注意：ANSI 序列的顺序是 **行在前，列在后**，所以是 `y;x`，和 Windows 的 `(x, y)` 参数顺序刚好相反。你调用时仍然传 `(x, y)`，函数内部自动转。

### 隐藏光标

```cpp
void hideCursor() {
    cout << "\033[?25l";
}
```

### 设置颜色

ANSI 颜色用不同的转义码，而且可以同时设前景和背景色：

```cpp
// 前景色（文字颜色）
void setColor(int color) {
    // 颜色映射：把 Windows 颜色代码转为 ANSI
    switch (color) {
        case 2:  cout << "\033[32m"; break;  // 深绿 → ANSI 绿
        case 7:  cout << "\033[37m"; break;  // 白色
        case 10: cout << "\033[92m"; break;  // 亮绿（蛇头）
        case 12: cout << "\033[91m"; break;  // 亮红（食物）
        case 14: cout << "\033[93m"; break;  // 亮黄（分数）
        default: cout << "\033[0m";  break;  // 重置
    }
}
```

ANSI 常用颜色码参考：

| ANSI 码 | 效果 |
|---------|------|
| `\033[0m` | 重置所有样式 |
| `\033[31m` | 红色 |
| `\033[32m` | 绿色 |
| `\033[33m` | 黄色 |
| `\033[34m` | 蓝色 |
| `\033[37m` | 白色 |
| `\033[91m` | 亮红 |
| `\033[92m` | 亮绿 |
| `\033[93m` | 亮黄 |

### 延时

Mac/Linux 用 `usleep`（微秒级）：

```cpp
#include <unistd.h>
usleep(150000);   // 150 毫秒 = 150000 微秒
```

### 非阻塞键盘输入

这是 Mac/Linux 上最复杂的部分。需要把终端从"行缓冲模式"切到"原始模式"，让每次按键立即可读：

```cpp
#include <termios.h>
#include <unistd.h>
#include <fcntl.h>

// 设置终端为非阻塞原始模式
void initTerminal() {
    struct termios t;
    tcgetattr(STDIN_FILENO, &t);
    t.c_lflag &= ~(ICANON | ECHO);   // 关闭行缓冲和回显
    t.c_cc[VMIN] = 0;                // 最少读取 0 个字符（非阻塞）
    t.c_cc[VTIME] = 0;               // 不等待
    tcsetattr(STDIN_FILENO, TCSANOW, &t);
}

// 恢复终端为正常模式（游戏结束前一定要调）
void resetTerminal() {
    struct termios t;
    tcgetattr(STDIN_FILENO, &t);
    t.c_lflag |= (ICANON | ECHO);
    tcsetattr(STDIN_FILENO, TCSANOW, &t);
}

// 检测是否有按键
bool _kbhit() {
    struct timeval tv = {0, 0};
    fd_set fds;
    FD_ZERO(&fds);
    FD_SET(STDIN_FILENO, &fds);
    return select(STDIN_FILENO + 1, &fds, NULL, NULL, &tv) > 0;
}

// 读取一个字符
int _getch() {
    char c;
    return read(STDIN_FILENO, &c, 1) == 1 ? c : -1;
}
```

> 这些函数名故意取成 `_kbhit` 和 `_getch`，这样你的游戏代码不用改调用方式。

### 方向键

和 Windows 一样的逻辑——方向键也是多字节序列。在 Mac/Linux 终端中，方向键通常以 `\033[` 开头（三个字节），你读的时候要连续读：

```cpp
int key = _getch();
if (key == 27) {             // 27 = ESC = \033
    if (_getch() == 91) {    // 91 = '['
        key = _getch();
        // ↑65 ↓66 →67 ←68 (ANSI 方向码)
    }
}
```

## 跨平台建议

如果想让同一份代码在 Windows 和 Mac 上都能跑，可以用 `#ifdef` 来做条件编译：

```cpp
#ifdef _WIN32
    // Windows 的实现
#else
    // Mac / Linux 的实现
#endif
```

`student/main.cpp` 目前只提供了 Windows 的实现。Mac 用户可以参考上面的 ANSI 方案自行替换辅助函数，核心游戏逻辑（蛇的移动、碰撞检测等）是跨平台的，不需要改动。

## 局部刷新

无论哪个平台，一个铁则是**不要在游戏循环里清屏**。`system("cls")`（Windows）和 `system("clear")`（Mac/Linux）都会导致画面剧烈闪烁。

本实验的策略是只重绘**变化的部分**：擦旧蛇尾、画新蛇头、食物移动时重画、分数变化时重写。
