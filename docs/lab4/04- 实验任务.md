# 分层卡池抽卡系统

## 一、实验目标

在本实验中，你将在 lab3 简单抽卡系统的基础上，实现一套**分层卡池抽卡算法**。通过本实验你将掌握：

1. 分层卡池的设计思想和架构
2. 如何在现有模板代码基础上扩展新功能
3. 动态概率控制和状态管理的编程技巧
4. 多保底机制（反保底 / 软保底 / 硬保底 / 补偿池）的实现

## 二、背景知识

请先阅读 `02 - 抽卡算法介绍.md` 了解分层卡池的完整设计原理。

**核心思路**：将一次抽卡分解为两层：
- **第一层 selectPool()**：根据当前状态（总抽数、保底计数、连续低星次数）选择进入哪个子池
- **第二层 drawFromPool()**：在选定的子池中按不同概率随机抽取

### TODO 12 — GachaSystem::selectPool (GachaSystem.cpp)

**要做的事**：根据状态变量选择当前进入哪个子池。按以下优先级用 if 判断：

| 优先级 | 条件 | 返回值 |
|--------|------|--------|
| 最高 | `pityCounter_ >= HARD_PITY_LIMIT - 1`（即 ≥89） | `PoolType::PITY` |
| ↓ | `totalDraws_ < ANTIPITY_LIMIT`（即 <10） | `PoolType::ANTIPITY` |
| ↓ | `consecutiveR_ >= FRIENDLY_THRESHOLD`（即 ≥5） | `PoolType::FRIENDLY` |
| ↓ | `pityCounter_ >= SOFT_PITY_START`（即 ≥70） | `PoolType::FRIENDLY` |
| 最低 | 其余所有情况 | `PoolType::NORMAL` |

### TODO 13 — GachaSystem::drawFromPool (GachaSystem.cpp)

**要做的事**：根据选定的池类别，用不同概率随机抽取物品。用 `switch (pool)` 实现：

| 池类别 | SSR 概率 | SR 概率 | R 概率 | 实现方式 |
|--------|---------|---------|--------|---------|
| `PITY` | 100% | 0% | 0% | 直接 `return ssrPool_.drawRandom()` |
| `ANTIPITY` | 0% | 15% | 85% | `rand()%100 < 15` → SR，否则 R |
| `FRIENDLY` | 2% + 加成 | 28% | 70% - 加成 | 见下方详细说明 |
| `NORMAL` | 2% | 13% | 85% | `roll<2` → SSR，`roll<15` → SR，否则 R |

**FRIENDLY 池详细说明**：
- 计算额外加成：`int extra = (pityCounter_ >= 70) ? (pityCounter_ - 70 + 1) * 5 : 0`
- SSR 区间：`[0, 2+extra-1]`，即 `roll < 2 + extra`
- SR 区间：`[2+extra, 30+extra-1]`，即 `roll < 30 + extra`
- 其余为 R

### TODO 14 — GachaSystem::singleDraw (GachaSystem.cpp)

**要做的事**：执行一次完整的单抽流程（串联 selectPool 和 drawFromPool）。

6 个步骤：
1. `totalDraws_++`
2. `PoolType pool = selectPool()`
3. `Item item = drawFromPool(pool)`
4. 更新状态：SSR → 重置计数 / R → 递增计数 / SR → 递增保底、重置连续 R
5. `inventory_.add(item)`
6. `return item`
