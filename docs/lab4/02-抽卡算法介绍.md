# 分层卡池（Tiered Gacha Pool）

## 一、什么是分层卡池

分层卡池是目前主流抽卡游戏（如原神、崩坏星穹铁道等）广泛使用的一种抽卡算法架构。它的核心思想是将一次抽卡过程分解为多个层级，每一层负责不同的决策逻辑：

1. **第一层：确定卡池类别** — 根据玩家当前状态（抽数、保底计数、连续低星次数）选择进入哪个子池
2. **第二层：选择具体卡池** — 在选定的子池中按权重随机抽取具体奖品

这种设计将复杂的抽卡规则拆解为清晰的模块，便于调节概率、控制玩家体验，也更容易公示和验证。

## 二、四种卡池类别

分层卡池通常包含以下四种子池：

### 2.1 普通池（NORMAL）

**触发条件**：正常情况下的默认选择。抽数在反保底期后、软保底前，且未触发补偿机制。

**概率配置**：

| 稀有度 | 概率 |
|--------|------|
| SSR | 2% |
| SR | 13% |
| R | 85% |

### 2.2 反保底池（ANTIPITY）

**触发条件**：新卡池的前 N 抽（通常为 10 抽），用于控制玩家不能在前几抽就获得顶级道具，避免"欧皇"过度影响付费意愿。

**概率配置**：

| 稀有度 | 概率 |
|--------|------|
| SSR | 0%（完全不出）|
| SR | 15% |
| R | 85% |

### 2.3 补偿池（FRIENDLY）

**触发条件**：
- **连续低星补偿**：连续 5 次抽到 R 后触发，概率获得提升
- **软保底（Soft Pity）**：第 70 抽起进入软保底区间，SSR 概率逐步递增

**概率配置（动态）**：

| 稀有度 | 基础概率 | 软保底加成 |
|--------|---------|-----------|
| SSR | 2% | 每超 70 抽 +5% |
| SR | 28% | — |
| R | 70% | — |

软保底加成公式：`SSR概率 = 2% + (pityCounter - 70 + 1) × 5%`，在第 89 抽时 SSR 概率达到 `2% + 19×5% = 97%`。

### 2.4 保底池（PITY）

**触发条件**：连续 89 抽未出 SSR，第 90 抽必进保底池。

**概率配置**：

| 稀有度 | 概率 |
|--------|------|
| SSR | 100%（必出）|
| SR | 0% |
| R | 0% |

## 三、算法流程

```
单抽 singleDraw():
  │
  ├─ 1. totalDraws++
  │
  ├─ 2. selectPool() — 根据状态选择卡池类别
  │    ├─ pityCounter >= 89  →  PITY（保底池）
  │    ├─ totalDraws < 10    →  ANTIPITY（反保底池）
  │    ├─ consecutiveR >= 5  →  FRIENDLY（补偿池）
  │    ├─ pityCounter >= 70  →  FRIENDLY（软保底）
  │    └─ 否则              →  NORMAL（普通池）
  │
  ├─ 3. drawFromPool(pool) — 在选定的池中按概率抽取
  │    ├─ PITY:     100% SSR
  │    ├─ ANTIPITY: 85% R, 15% SR（无 SSR）
  │    ├─ FRIENDLY: 动态概率（软保底加成）
  │    └─ NORMAL:   2% SSR, 13% SR, 85% R
  │
  ├─ 4. 更新状态
  │    ├─ 抽到 SSR → pityCounter = 0, consecutiveR = 0
  │    ├─ 抽到 R   → pityCounter++, consecutiveR++
  │    └─ 抽到 SR  → pityCounter++, consecutiveR = 0
  │
  └─ 5. 入库并返回物品
```

## 四、状态变量说明

| 变量 | 含义 | 作用 |
|------|------|------|
| `totalDraws_` | 总抽数 | 判断是否在反保底期 |
| `pityCounter_` | 距离上次 SSR 的连续抽数 | 判断软保底/硬保底触发 |
| `consecutiveR_` | 连续抽到 R 的次数 | 判断是否触发补偿池 |

## 五、与简单抽卡算法的对比

| 维度 | 简单算法（lab3）| 分层卡池算法 |
|------|---------------|------------|
| 卡池结构 | 按稀有度分 3 个池 | 按行为策略分 4 个池 |
| 概率 | 固定概率 | 动态概率 + 状态驱动 |
| 保底 | 仅硬保底（90 抽）| 反保底 + 软保底 + 补偿 + 硬保底 |
| 模块化 | 所有逻辑在 singleDraw 中 | selectPool / drawFromPool 分离 |
| 可调节性 | 改概率需改核心逻辑 | 每个子池独立可调 |
