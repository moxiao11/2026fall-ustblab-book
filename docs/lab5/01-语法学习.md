## 一、文件读写操作

### 1.1 二进制文件写入

`WavWriter` 需要将 PCM 音频数据保存为 `.wav` 文件（TODO 6）。WAV 是二进制格式，不能直接用 `<<` 写入文本。

**打开文件**：

```cpp
#include <fstream>

std::ofstream file(filename, std::ios::binary);
if (!file) {
    return false;  // 打开失败
}
```

关键点：
- `std::ofstream`：输出文件流（output file stream）
- `std::ios::binary`：**必须指定**，否则在 Windows 下写入 `0x0A` 时会被自动替换为 `0x0D 0x0A`
- 检查 `!file` 判断文件是否成功打开

**写入原始字节**：

```cpp
file.write(指针, 字节数);
```

第一个参数是 `const char*` 类型。如果要写入非 char 数据（如 `int16_t` 数组），需要用 `reinterpret_cast`：

```cpp
file.write(reinterpret_cast<const char*>(samples.data()), dataSize);
```

- `samples.data()` 返回 `const int16_t*`
- `reinterpret_cast<const char*>` 将 int16_t 指针强制转成 char 指针（不改变数据，只改变读取方式）
- `dataSize` = 采样数量 × `sizeof(int16_t)`

### 1.2 小端序字节写入

WAV 文件要求使用**小端序**（Little-Endian）：低位字节在前，高位字节在后。这是 TODO 7 的内容。

以 `int16_t` 值 `0x1234` 为例：

| 小端序 | 大端序 |
|--------|--------|
| byte[0] = 0x34 | byte[0] = 0x12 |
| byte[1] = 0x12 | byte[1] = 0x34 |

**writeInt16 实现**：

```cpp
void writeInt16(std::ofstream& file, int value) const {
    const char bytes[2] = {
        static_cast<char>(value & 0xff),         // 低 8 位
        static_cast<char>((value >> 8) & 0xff)   // 高 8 位
    };
    file.write(bytes, 2);
}
```

**writeInt32 实现**：

```cpp
void writeInt32(std::ofstream& file, int value) const {
    const char bytes[4] = {
        static_cast<char>(value & 0xff),
        static_cast<char>((value >> 8) & 0xff),
        static_cast<char>((value >> 16) & 0xff),
        static_cast<char>((value >> 24) & 0xff)
    };
    file.write(bytes, 4);
}
```

**writeString 实现**（已给出，无需修改）：

```cpp
void writeString(std::ofstream& file, const std::string& text) const {
    file.write(text.c_str(), static_cast<std::streamsize>(text.size()));
}
```

### 1.3 WAV 文件头字段的计算

以下字段在 `save()` 中需要计算（TODO 6）：

| 字段 | 含义 | 计算方式 |
|------|------|----------|
| `byteRate` | 每秒字节数 | `sampleRate × channels × bitsPerSample / 8` |
| `blockAlign` | 每帧字节数 | `channels × bitsPerSample / 8` |
| `dataSize` | PCM 数据总字节 | `samples.size() × sizeof(int16_t)` |
| `chunkSize` | 文件剩余大小 | `36 + dataSize` |

WAV 写入顺序：

```
"RIFF" → chunkSize → "WAVE"
"fmt " → 16 → PCM(1) → channels → sampleRate → byteRate → blockAlign → bitsPerSample
"data" → dataSize → [PCM 采样数据...]
```

## 二、关联容器 — std::map

### 2.1 为什么用 map

`FrequencyTable` 需要根据音名字符串（如 `"C"`、`"G#"`）查找对应的半音编号（0~11）。这是一个典型的"键 → 值"查找场景，用 `std::map` 实现。

```cpp
#include <map>

class FrequencyTable {
private:
    std::map<std::string, int> semitoneMap;  // 音名 → 半音编号
};
```

### 2.2 在初始化列表中填充

```cpp
FrequencyTable::FrequencyTable()
    : semitoneMap({
          {"C", 0}, {"C#", 1}, {"D", 2}, {"D#", 3},
          {"E", 4}, {"F", 5}, {"F#", 6}, {"G", 7},
          {"G#", 8}, {"A", 9}, {"A#", 10}, {"B", 11}
      }) {
}
```

语法说明：
- 外层 `{ }` 是 map 的 initializer_list
- 内层每个 `{"C", 0}` 是一个 `std::pair<std::string, int>`
- C++ 会自动推导类型

### 2.3 使用 find() 查找

`std::map::find()` 返回一个**迭代器**：
- 找到 → 指向该键值对，可通过 `it->second` 获取值
- 没找到 → 等于 `semitoneMap.end()`

```cpp
auto it = semitoneMap.find(pitch);
if (it == semitoneMap.end()) {
    return 0.0;  // 无效音名
}
int semitone = it->second;  // 半音编号
```

## 三、类型转换与容器操作

### 3.1 static_cast

C++ 提供了四种类型转换，`static_cast` 是最常用的一种，在编译期检查转换是否合法。

本实验中出现的用法：

```cpp
// 浮点 → 整数（截断小数部分）
int sampleCount = static_cast<int>(duration * SAMPLE_RATE);

// 整数 → 字符（用于字节写入）
static_cast<char>(value & 0xff);

// 指针 → 不同解释方式（WavWriter 中小端序写入）
file.write(reinterpret_cast<const char*>(samples.data()), dataSize);
```

注意 `static_cast` vs `reinterpret_cast`：
- `static_cast`：合理的类型转换（如 int→char），编译器会检查
- `reinterpret_cast`：重新解释二进制位（如 int16_t* → char*），危险但必要

### 3.2 固定宽度整数类型 — int16_t

```cpp
#include <cstdint>
```

| 类型 | 宽度 | 范围 |
|------|------|------|
| `int16_t` | 16 bit（2 字节） | -32768 ~ 32767 |
| `int32_t` | 32 bit（4 字节） | -2147483648 ~ 2147483647 |
| `uint16_t` | 16 bit 无符号 | 0 ~ 65535 |

本实验使用 `int16_t` 作为采样类型，因为 CD 音质的 WAV 文件使用 16 bit PCM 编码。

波形值（`double`，范围 -1.0 ~ 1.0）转为 `int16_t`：

```cpp
samples.push_back(static_cast<int16_t>(value * 32767));
```

`32767` 是 `int16_t` 的最大正数值，乘以此数将浮点波形映射到整数范围。

### 3.3 std::vector 基本操作

```cpp
#include <vector>

std::vector<int16_t> samples;           // 创建空 vector
samples.push_back(value);               // 追加一个元素到末尾
int n = samples.size();                 // 获取元素数量
const int16_t* ptr = samples.data();    // 获取底层数组指针（用于 file.write）
```

遍历方式：

```cpp
// 范围 for（只读）
for (int16_t sample : samples) { ... }

// 范围 for（可修改）
for (int16_t& sample : samples) { sample = ...; }
```

## 四、数值算法与数学函数

### 4.1 std::min 与 std::max

```cpp
#include <algorithm>

double x = std::min(a, b);  // 返回较小值
double y = std::max(a, b);  // 返回较大值
```

在 ADSR 包络（TODO 2）中的典型用法——限制各阶段时长不超过音符总时长：

```cpp
double attack = std::min(config.getAttack(), duration * 0.25);
double release = std::min(config.getRelease(), duration * 0.35);
```

### 4.2 std::clamp

```cpp
#include <algorithm>

value = std::clamp(value, -1.0, 1.0);  // 限制在 [-1.0, 1.0] 内
```

等价于：

```cpp
if (value < -1.0) value = -1.0;
if (value > 1.0)  value = 1.0;
```

在音频合成中，clamp 防止采样值超出合法范围导致爆音。

### 4.3 std::pow 与 std::sin

```cpp
#include <cmath>

double result = std::pow(2.0, 3.0);      // 2.0^3.0 = 8.0
double wave = std::sin(2.0 * PI * f * t);  // 正弦波
```

十二平均律频率公式（TODO 1）：

```cpp
frequency = 440.0 * std::pow(2.0, (midiNumber - 69) / 12.0);
```

谐波叠加（TODO 3）：

```cpp

       + 0.45 * std::sin(2.0 * PI * frequency * 2.0 * t)
       + 0.22 * std::sin(2.0 * PI * frequency * 3.0 * t)
       + 0.10 * std::sin(2.0 * PI * frequency * 4.0 * t);
```

### 4.4 std::abs


## 五、其他琐碎知识


### 5.1 使用 substr() 拆分字符串

```cpp
std::string noteName = "C4";
std::string pitch = noteName.substr(0, noteName.size() - 1);  // "C"（从位置0取len-1个字符）
int octave = noteName.back() - '0';  // '4' - '0' = 4
```

- `substr(起始位置, 长度)`：截取子串
- `back()`：返回最后一个字符
- `'4' - '0'`：字符转数字的技巧（'0' 的 ASCII 码是 48，'4' 是 52，相减得 4）
