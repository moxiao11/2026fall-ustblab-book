# 测试文件介绍


相信写测试代码是最讨厌的一个环节，因为只需要提交到CG即可看代码是否正确，或者询问AI，但是这一部分仍旧是程序员不可或缺的能力。

---

## 1. 测试文件的结构

本次实验的测试代码分成三个文件：

```text
student/
  test.h       -> 声明测试函数
  test.cpp     -> 编写具体测试内容
  main.cpp     -> 程序入口，按顺序调用测试函数
```

### test.h：声明有哪些测试

```cpp
#ifndef TEST_H
#define TEST_H

void testConstructor();
void testAccess();
void testZeros();
void testOnes();
void testArange();
void test3D();
void testFull();
void testFill();
void testReshape();
void testSum();

#endif
```

`test.h` 只负责告诉其他文件：这里有哪些测试函数可以调用。

### main.cpp：调用所有测试

```cpp
#include "test.h"
#include <iostream>
using namespace std;

int main() {
    cout << "========== Lab3 Stage 1: NdArray Core ==========" << endl;
    cout << endl;

    testConstructor();
    testAccess();
    testZeros();
    testOnes();
    testArange();
    test3D();
    testFull();
    testFill();
    testReshape();
    testSum();

    cout << "All tests passed!" << endl;
    return 0;
}
```

`main.cpp` 不负责判断每个功能对不对，它只负责按顺序运行测试。真正的检查逻辑写在 `test.cpp` 中。

### test.cpp：用 assert 检查结果

`test.cpp` 会创建 `NdArray` 对象，调用它的成员函数，然后用 `assert` 判断结果是否正确。

```cpp
#include "NdArray.h"
#include "test.h"
#include <cassert>
#include <cmath>
#include <vector>
using namespace std;
```

其中：

- `<cassert>` 提供 `assert(...)`。
- `<cmath>` 提供 `fabs(...)`，用于比较浮点数。
- `<vector>` 用来表示 shape 和下标。

---

## 2. assert 是什么

`assert` 是一种最简单的自动测试方式。

```cpp
assert(a.size() == 6);
```

含义是：程序运行到这里时，`a.size()` 必须等于 `6`。

如果条件成立，程序继续运行。如果条件不成立，程序会立刻停止，并显示出错的位置。

所以：

```cpp
assert(实际结果 == 期望结果);
```

就是测试中最常见的写法。

## 3. test.cpp 中每个测试测什么

### testConstructor

```cpp
void testConstructor() {
    NdArray<double> a({2, 3});
    assertShape(a, {2, 3});
    assert(a.size() == 6);
    assert(a.ndim() == 2);
}
```

这个测试检查构造函数和基本属性是否正确：

- `NdArray<double> a({2, 3})` 能否创建一个 2 行 3 列数组。
- `shape()` 是否返回 `{2, 3}`。
- `size()` 是否返回总元素个数 `6`。
- `ndim()` 是否返回维度数 `2`。

### testAccess

```cpp
void testAccess() {
    NdArray<int> b({2, 3});
    int val = 1;
    for (int i = 0; i < 2; i++) {
        for (int j = 0; j < 3; j++) {
            b({i, j}) = val++;
        }
    }

    assert(b({0, 0}) == 1);
    assert(b({0, 1}) == 2);
    assert(b({0, 2}) == 3);
    assert(b({1, 0}) == 4);
    assert(b({1, 1}) == 5);
    assert(b({1, 2}) == 6);
}
```

这个测试检查 `operator()` 是否能正确读写二维数组元素。

它先把数组填成：

```text
1 2 3
4 5 6
```

然后逐个位置检查读出来的值是否正确。

### testZeros

```cpp
void testZeros() {
    auto z = NdArray<double>::zeros({3, 4});
    assertShape(z, {3, 4});
    assert(z.size() == 12);
    assert(z.ndim() == 2);

    for (int i = 0; i < 3; i++) {
        for (int j = 0; j < 4; j++) {
            assert(z({i, j}) == 0.0);
        }
    }
}
```

这个测试检查 `zeros` 是否正确：

- 创建出来的数组 shape 是否是 `{3, 4}`。
- 总元素个数是否是 `12`。
- 维度数是否是 `2`。
- 每个元素是否都是 `0`。

### testOnes

```cpp
void testOnes() {
    auto o = NdArray<double>::ones({2, 2});
    assertShape(o, {2, 2});
    assert(o.size() == 4);
    assert(o.ndim() == 2);

    for (int i = 0; i < 2; i++) {
        for (int j = 0; j < 2; j++) {
            assert(o({i, j}) == 1.0);
        }
    }
}
```

这个测试检查 `ones` 是否正确：

- 创建出来的数组 shape 是否是 `{2, 2}`。
- 总元素个数是否是 `4`。
- 每个元素是否都是 `1`。

### testArange

```cpp
void testArange() {
    auto r1 = NdArray<int>::arange(0, 10, 1);
    assertShape(r1, {10});
    assert(r1.size() == 10);
    for (int i = 0; i < 10; i++) {
        assert(r1({i}) == i);
    }

    auto r2 = NdArray<int>::arange(0, 10, 2);
    assertShape(r2, {5});
    assert(r2.size() == 5);
    for (int i = 0; i < 5; i++) {
        assert(r2({i}) == i * 2);
    }

    auto r3 = NdArray<double>::arange(0.0, 1.0, 0.2);
    assertShape(r3, {5});
    assert(r3.size() == 5);
    for (int i = 0; i < 5; i++) {
        assertClose(r3({i}), i * 0.2);
    }
}
```

这个测试检查 `arange(start, end, step)` 是否正确生成一维数组：

- `arange(0, 10, 1)` 应该生成 `0, 1, 2, ..., 9`。
- `arange(0, 10, 2)` 应该生成 `0, 2, 4, 6, 8`。
- `arange(0.0, 1.0, 0.2)` 应该生成 `0.0, 0.2, 0.4, 0.6, 0.8`。

浮点数不能直接用 `==` 比较，所以这里使用：

```cpp
void assertClose(double actual, double expected) {
    assert(fabs(actual - expected) < 1e-9);
}
```

只要两个数的差足够小，就认为它们相等。

### test3D

```cpp
void test3D() {
    NdArray<int> c({2, 2, 2});
    assertShape(c, {2, 2, 2});
    assert(c.size() == 8);
    assert(c.ndim() == 3);

    c({0, 0, 0}) = 1;
    c({0, 0, 1}) = 2;
    c({0, 1, 0}) = 3;
    c({0, 1, 1}) = 4;
    c({1, 0, 0}) = 5;
    c({1, 0, 1}) = 6;
    c({1, 1, 0}) = 7;
    c({1, 1, 1}) = 8;

    assert(c({0, 0, 0}) == 1);
    assert(c({0, 0, 1}) == 2);
    assert(c({0, 1, 0}) == 3);
    assert(c({0, 1, 1}) == 4);
    assert(c({1, 0, 0}) == 5);
    assert(c({1, 0, 1}) == 6);
    assert(c({1, 1, 0}) == 7);
    assert(c({1, 1, 1}) == 8);
}
```

这个测试检查三维数组是否能正确工作：

- shape 是否是 `{2, 2, 2}`。
- size 是否是 `8`。
- ndim 是否是 `3`。
- 三维下标访问是否能正确映射到底层一维数据。

### testFull

```cpp
void testFull() {
    auto f = NdArray<double>::full({2, 3}, 3.14);
    assertShape(f, {2, 3});
    assert(f.size() == 6);
    assert(f.ndim() == 2);
    for (int i = 0; i < 2; i++) {
        for (int j = 0; j < 3; j++) {
            assertClose(f({i, j}), 3.14);
        }
    }

    auto f2 = NdArray<int>::full({4}, 7);
    assertShape(f2, {4});
    assert(f2.size() == 4);
    for (int i = 0; i < 4; i++) {
        assert(f2({i}) == 7);
    }
}
```

这个测试检查 `full(shape, value)` 是否正确：

- `full({2, 3}, 3.14)` 创建 2×3 数组，每个元素都是 3.14。
- `full({4}, 7)` 创建一维数组，每个元素都是 7。

### testFill

```cpp
void testFill() {
    NdArray<int> a({2, 3});
    a.fill(42);
    for (int i = 0; i < 2; i++) {
        for (int j = 0; j < 3; j++) {
            assert(a({i, j}) == 42);
        }
    }

    a.fill(-1);
    for (int i = 0; i < 2; i++) {
        for (int j = 0; j < 3; j++) {
            assert(a({i, j}) == -1);
        }
    }
}
```

这个测试检查 `fill(value)` 是否正确：

- 先 fill(42)，检查所有元素是否变成 42。
- 再 fill(-1)，检查能否再次覆盖为 -1。

### testReshape

```cpp
void testReshape() {
    NdArray<int> a({2, 3});
    for (int i = 0; i < 6; i++) {
        a({i / 3, i % 3}) = i;
    }

    auto r = a.reshape({3, 2});
    assertShape(r, {3, 2});
    assert(r.size() == 6);
    assert(r.ndim() == 2);
    assert(r({0, 0}) == 0);
    assert(r({0, 1}) == 1);
    assert(r({1, 0}) == 2);
    assert(r({1, 1}) == 3);
    assert(r({2, 0}) == 4);
    assert(r({2, 1}) == 5);

    auto r2 = a.reshape({6});
    assertShape(r2, {6});
    assert(r2.size() == 6);
    assert(r2.ndim() == 1);
    for (int i = 0; i < 6; i++) {
        assert(r2({i}) == i);
    }
}
```

这个测试检查 `reshape(new_shape)` 是否正确：

- 把 2×3 变成 3×2，检查数据顺序不变（row-major）。
- 把 2×3 变成一维 `{6}`，检查数据顺序不变。

### testSum

```cpp
void testSum() {
    NdArray<int> a({2, 3});
    for (int i = 0; i < 6; i++) {
        a({i / 3, i % 3}) = i;
    }
    assert(a.sum() == 15);

    auto z = NdArray<double>::zeros({3, 3});
    assertClose(z.sum(), 0.0);

    auto o = NdArray<int>::ones({2, 2});
    assert(o.sum() == 4);

    auto f = NdArray<double>::full({5}, 2.0);
    assertClose(f.sum(), 10.0);
}
```

这个测试检查 `sum()` 是否正确：

- `0+1+2+3+4+5 = 15`。
- zeros 数组求和为 0。
- ones 数组 2×2 求和为 4。
- full({5}, 2.0) 求和为 10.0。

## 4. 测试文件如何编写

编写一个测试函数时，可以按照下面的步骤：

1. 创建一个对象。
2. 调用要测试的函数。
3. 写出期望结果。
4. 用 `assert` 比较实际结果和期望结果。

例如，要测试一个 2 行 2 列的数组：

```cpp
void testExample() {
    NdArray<int> a({2, 2});

    a({0, 0}) = 10;
    a({0, 1}) = 20;
    a({1, 0}) = 30;
    a({1, 1}) = 40;

    assert(a({0, 0}) == 10);
    assert(a({0, 1}) == 20);
    assert(a({1, 0}) == 30);
    assert(a({1, 1}) == 40);
}
```

如果要把这个测试加入项目，需要做三件事：

1. 在 `test.cpp` 里实现 `testExample()`。
2. 在 `test.h` 里声明 `void testExample();`。
3. 在 `main.cpp` 里调用 `testExample();`。

---

## 5. 写测试时的建议

- 一个测试函数只测一类功能，比如构造、访问、zeros、ones。
- 不要只打印结果，要用 `assert` 自动判断对错。
- 测数组内容时，不只测第一个元素，最好把关键位置都测到。
- 测二维或三维访问时，要覆盖不同下标位置。
- 测浮点数时不要直接用 `==`，应使用 `fabs(actual - expected) < 1e-9` 这样的近似比较。

如果所有 `assert` 都通过，程序会正常运行到最后并输出：

```text
All tests passed!
```

如果某个 `assert` 失败，程序会立刻停止，说明对应功能还没有实现正确。
