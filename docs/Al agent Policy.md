# 程序设计课程 AI Agent 使用规范

## 一、目的

随着 ChatGPT、GitHub Copilot、Claude、Cursor 等 AI Agent 工具的发展，程序设计学习方式正在发生变化。

本课程允许学生在合理范围内使用 AI Agent 辅助学习，但其目的应是：

1. 帮助理解程序设计概念；
2. 辅助调试和分析错误；
3. 提高代码阅读与改进能力；
4. 培养学生与 AI 协作完成程序设计任务的能力。

AI Agent 可以作为学习助手，但不能替代学生本人的思考、设计、编码与总结。

---

## 二、适用范围

本规范适用于本课程中的以下环节：

- 课堂练习；
- 实验作业；
- 课程项目；
- 课后学习；
- 代码调试；
- 复习与自学。

对于考试、测验、限时上机等考核环节，是否允许使用 AI Agent，应以教师当次说明为准。

---

## 三、允许使用 AI Agent 的场景

### 1. 概念理解

学生可以使用 AI Agent 辅助理解程序设计中的基本概念，例如：

- 变量、表达式、语句；
- 顺序结构、选择结构、循环结构；
- 函数、参数、返回值；
- 数组、字符串、指针、引用；
- 结构体、类和对象；
- 递归、文件操作、简单数据结构、算法等。
- 包括但不限于上面的

示例：

```text
请用通俗的语言解释 C++ 中指针和引用的区别。
```
### 2. 代码调试

可以将自己的代码提交给 AI Agent，请其帮助分析错误原因。

允许的提问方式：
```text
这是我写的代码，运行结果不正确，请帮我分析可能的问题。
```
不推荐的提问方式：
```text
直接帮我写一个满分答案。
```
### 3. 代码解释

可以让 AI Agent 解释已有代码的含义，例如：

- 解释某一段代码的执行流程；
- 分析函数的作用；
- 说明某个变量为什么这样变化；
- 讲解递归调用过程。

### 4. 代码优化建议

学生可以请 AI Agent 对自己已经完成的代码提出改进建议，例如：

- 是否可以减少重复代码；
- 是否可以提高可读性；
- 是否可以使用函数封装；
- 是否存在潜在的边界情况问题。

### 5. 学习资料辅助

可以使用 AI Agent 生成学习提纲、复习笔记、测试样例和练习题。

例如：
```text
请帮我整理一份 C++ 函数部分的复习提纲。
```

## 四、代码责任原则

学生都必须对最终提交的代码负责。

学生需要能够回答以下问题：
```text
这段代码的整体思路是什么？
每个函数的作用是什么？
关键变量的含义是什么？
程序如何处理边界情况？
如果输入发生变化，程序是否仍然正确？
代码中是否存在潜在错误？
包括但不限于以上问题
```
如果学生无法解释自己提交的代码，即使代码可以运行，也可能被视为不合格提交。
